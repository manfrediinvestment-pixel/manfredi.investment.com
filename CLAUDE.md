# Reglas de trabajo con git

- Nunca hagas commits ni cambios directamente en la rama `main`.
- Para cualquier tarea, primero creá una rama nueva y descriptiva (ej: `fix/nombre-del-arreglo`).
- Trabajá y commiteá en esa rama, y avisá cuando esté lista para que el usuario la revise antes de mergear.
- Nunca hagas merge a `main` sin que el usuario lo confirme explícitamente.
